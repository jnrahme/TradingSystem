Most of the storage gap is not runtime code

The old repo carries hundreds of megabytes of git history, worktrees, data files, backtests, logs, and accumulated docs. The new repo is much smaller because it is not dragging those artifacts along yet.

The capability gap is still real

The old repo has many more strategies, scripts, workflows, safety rules, and operational helpers. We need to bring over the good parts, but only through the new platform boundaries.

Legacy Inventory

Strategies

The legacy repo contains iron condor, momentum, REIT, Rule One options, VIX mean reversion, and other strategy modules. The first extracted plugin is already the legacy iron condor path.

Operational Scripts

The legacy repo has more than one hundred scripts across trading, sync, verification, research, content, and maintenance. The new platform should collapse these into a smaller product-grade CLI surface.

Workflows

The old repo has 90 GitHub workflows spanning runtime trading, alerts, emergency actions, CI, content, docs, and experiments. The new repo should keep only a much smaller workflow set after logic is moved into code.
